Features

Excel & CSV Parsing

Reads and interprets your Snowflake schema from Excel or CSV files with robust validation.

Talend XML Generation

Automatically maps Snowflake types to Talend types and generates Talend-compatible XML schemas.

Instant Output & Download

Generates XML instantly, lets you copy or download the result, and provides a sample file for reference.

Analysis & Summary

Shows parsed columns summary, record count, file size, file name, and real processing time for every upload.

User-Friendly UI

Modern, responsive design with error/debug tab, and clear instructions.

Robust Error Handling

Shows clear error/debug info for upload issues, schema problems, and file size limits (max 5MB enforced).

How to Use

  1. Download the sample file from the Navbar for reference.
  2. Prepare your Excel or CSV file with the required columns and format as described below.
  3. Click the "Upload Your Excel File" button above and select your file.
  4. Wait for SchemaBridge to parse and generate the Talend XML schema.
  5. Download and validate the XML in Talend for compatibility.

Input File Requirements

  • Required columns: Column_Name and Column_Type (case-sensitive).
  • Accepted file formats: .xlsx (Excel) or .csv (CSV).
  • Maximum file size: 5MB
  • Each row must have both Column_Name and Column_Type filled in.

Column Details

  • Column_Name: The name of the column as it should appear in Talend.
  • Column_Type: The Snowflake data type (e.g., VARCHAR(50), NUMBER, BOOLEAN).

Common Mistakes to Avoid

  • Do not rename or misspell the required columns (Column_Name, Column_Type).
  • Do not leave any required cell blank in a row.
  • Do not use unsupported file formats (only .xlsx and .csv are allowed).
  • Do not include extra header rows or merged cells.
  • Do not use formulas or special characters in column names.

Tips

  • Refer to the sample file for the correct structure and formatting.
  • Use plain text values for all cells.
